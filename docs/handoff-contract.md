# Terraform to Ansible Handoff Contract

## Intent
Define the canonical payload passed from Terraform provisioning to Ansible operational workflows.

## Preconditions
- Terraform run has completed and produced required outputs.
- Target and portal selectors have been validated.

## Payload Schema
The payload is represented by `terraable.contract.HandoffPayload`.

Required fields:
- `environment_name`: Friendly environment identifier.
- `terraform_run_id`: Source Terraform run identifier.
- `target_platform`: One of `openshift`, `aws`, `azure`, `okd`, or `local-lab`.
- `portal_impl`: One of `rhdh` or `backstage`.
- `security_profile`: One of `baseline` or `strict`.
- `connection`: Connection details object:
  - `ansible_inventory_group`
  - `ssh_user`
  - `ssh_port`
  - `api_endpoint`
- `metadata`: Free-form key/value metadata.

## Validation Rules
- Target and portal selectors are validated for enum correctness before workflow dispatch.
- Target-specific combination rules are applied only when an explicit matrix restriction is defined.

## Outputs
- Runtime variable dictionary generated by `to_runtime_vars()`.
- Payload is suitable for AAP/AWX runtime variables.

## Failure Modes and Remediation
- Unknown enum values raise a `ValueError`: confirm selectors provided by control-plane UI.
- Missing connection keys raise `KeyError`: ensure Terraform outputs include connection block.
